## Project Conventions

### Instruction Files

| File | Notes |
| --- | --- |
| `AGENTS.md` | Canonical project instructions; requires loading the Mastra skill first and points KIN lookups to `.atl/docs-routing.yaml` |
| `CLAUDE.md` | Lightweight pointer back to `AGENTS.md` |

### Technical Conventions

- Use Mastra structure under `src/mastra/{agents,workflows,tools,constants,scorers}`.
- Treat `.atl/docs-routing.yaml` as canonical for external-library documentation routing.
- Prefer TypeScript in ESM/bundler mode (`type: module`, `moduleResolution: bundler`, `strict: true`).
- Runtime is mixed Bun/Node oriented: `bun.lock` exists, but `package.json` engines require Node `>=22.13.0`.

## Loading Protocol For Sub-Agents

1. Try Engram first: search `skill-registry` in project memory.
2. Fallback to reading `.atl/skill-registry.md`.
3. Load relevant project skills first (`mastra` for any Mastra code).
4. If the task touches external libraries, activate `kin` and follow `.atl/docs-routing.yaml` before other doc sources.
5. Then read any relevant project instruction files, especially `AGENTS.md`.
